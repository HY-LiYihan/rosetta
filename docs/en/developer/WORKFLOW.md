# Workflow

Last updated: 2026-05-08

Development checklist:

1. Read repository instructions.
2. Keep behavior changes small and documented.
3. Update `docs/CHANGELOG.md` for code or behavior changes.
4. Update `README.md` when user-facing behavior changes.
5. Run compile checks, relevant tests, and MkDocs strict build.
6. Pull before push when pushing is requested.

Documentation review should use three strict roles: first-time user, PLM/NLP researcher, and Rosetta maintainer.
